# Sift — роадмеп до первых денег

*Составлено 10 июля 2026. Все решения зафиксированы в разговоре. Это живой документ — возвращайся к нему между сессиями.*

---

## Зафиксированные решения (не переоткрывать)

**Монетизация.** One-time, один тир: **€10 за 20 готовых пакетов** (тейлоренное резюме + сопроводительное) по подходящим вакансиям. Поиск, скоринг, b2b-оценка и просмотр разбора — бесплатны (крючок + дистрибуция). Подписка-мониторинг — позже, поверх, когда first-payers докажут спрос.

**Позиционирование.** Killer-фича — `b2b_eligible` (подходит ли вакансия под контракт через ИП). Метка `yes/maybe/no` на каждой вакансии + сортировка (b2b вторичный ключ). Никакого обещания «полного охвата» — обещаем качество оценки и экономию времени.

**Охват.** Умеренный, бесплатными средствами: чистка ATS-слагов + location fix + multi-role поиск (ИИ предлагает роли из резюме, юзер выбирает галочками). **Прокси НЕ нужны.** Pool-read — отложенный рычаг, не для старта.

**Что уже сделано и в main.** Точность фильтров: реальная дата вакансии (fix #2), b2b в сортировке (fix #3), строгий фильтр свежести JobSpy (fix #1). Подтверждено живым прогоном (счётчик freshness в логе). Merge-коммит 77e7517.

## Диагноз с последнего живого прогона

Прогон «Technical Support / Poland / 24h» вернул **0 живых вакансий** + 1 из WWR. Причины, все чинибельные:
- Один keyword вместо нескольких ролей (в прототипе было 10 ролей — в SaaS сузили до одного поля).
- Мёртвые ATS-слаги: doist/zapier/lever отдают 404 (в логе видно).
- Троттлинг LinkedIn на узком запросе.

**Вывод:** охват — не «потом», а первый блок работы. Без него платить не за что (нет вакансий → нет 20 пакетов → нет €10).

## ФАЗА 1 — Охват (13–26 июля, ~2 недели)

**Цель:** прогон стабильно возвращает достаточно вакансий, чтобы 20 пакетов было из чего собрать. Не «полнота», а «достаточно».

**Задачи (по порядку, все дешёвые, без прокси):**

1. **Чистка ATS-слагов.** Убрать мёртвые (doist/zapier/lever — 404), добавить живые remote-first компании. Проверять по логу каждый слаг. *S, безопасно.*

2. **Location fix.** Разворачивать регион («European Union») в список стран; чинить невалидные локации (баг `scraped=1`). В твоих issues. *S–M.*

3. **Multi-role поиск (фича автоключевых слов).** ИИ читает резюме → предлагает роли (не навыки!) для поиска → юзер выбирает галочками → каждая роль × локация = запрос, с дедупом. Извлекать РОЛИ (support engineer, integration specialist), НЕ навыки (SQL, MQTT, Azure — они для ATS-оценки, не для поиска). На старте — топ 5–8 ролей, предвыбрано 3–4. *M, переиспользует существующий LLM-парсинг CV + chips-input.*

**Метрика успеха:** типичный прогон по нише возвращает 20–40 релевантных вакансий (не ноль). Проверять счётчик freshness — не режет ли фильтр слишком агрессивно.

**Критерий остановки:** если после всех трёх задач прогон по типичному запросу ниши всё равно даёт <10 вакансий стабильно — значит бесплатных источников не хватает, и надо либо (а) честно пересмотреть обещание «20 пакетов» вниз, либо (б) впервые задуматься о платных источниках. НЕ раньше этой точки.

**НЕ делать в этой фазе:** прокси, pool-read, группу «навыки» в поиске, кабинет, подписку, «два варианта поиска» в UI.

## ФАЗА 0 — Проверка оплаты руками (параллельно/сразу после, 2 недели)

*Называется «Фаза 0», потому что это главный гейт — до любых вложений в автоматизацию проверяем, платит ли ниша вообще.*

**Цель:** 5 платящих за €10. Ноль нового кода сверх Фазы 1.

**Сначала — grunt-тест (полдня, критично):** возьми 15–20 реальных постов вакансий из целевых каналов, прогони через пайплайн, посмотри — на скольких `b2b_eligible` даёт ОСМЫСЛЕННЫЙ вердикт, а не «непонятно». Если на большинстве осмысленный — оффер про b2b жив. Если «непонятно» — оффер сдвигается на «оценю фит + соберу пакеты», b2b как бонус.

**Оффер (пример, адаптируй):**
> «Инженер, сам искал remote под ИП. Собрал инструмент: по твоему CV находит вакансии, оценивает, подходят ли они под B2B/ИП, и собирает готовые пакеты отклика — резюме + письмо, без выдуманных навыков. 20 пакетов за €10. Пришли CV — верну за 24 часа.»

**Где искать 10–15 человек (ПРОВЕРЬ каждый канал сам — существует ли, можно ли писать, там ли ниша):**
- Чаты релокантов/контракторов с ИП: армянские relocate-чаты (ты сам там), грузинские/сербские/черногорские IT-хабы, кипрская IT-тусовка.
- vas3k.club — пост «сделал инструмент, ищу бета-юзеров за деньги».
- Твой LinkedIn + личные сообщения знакомым из Axxonsoft/LAB, кто ищет.

**Механика:** 3 публичных бесплатных разбора как контент («вот вакансия, вот почему туда НЕ надо через ИП») → в конце «делаю пакеты за €10».

**Оплата:** ссылка Lemon Squeezy / Paddle (MoR, закрывает EU VAT). Час настройки.

**Метрика успеха:** ≥5 оплат из ~60–80 диалогов (конверсия ниши цено-чувствительная, закладывай 5–8%, не 12%). ≥2 вернулись за вторым.

**Критерий остановки:** <3 оплат после 60+ целевых диалогов → ниша не платит за это в текущей форме. Тогда живой остаток — контент-канал «разборы вакансий под ИП» + консультации, не SaaS.

## ФАЗА 2 — Self-serve + checkout в продукте (после ≥5 ручных оплат, 3 недели)

**Цель:** покупатель проходит путь без тебя. 15 суммарных оплат.

**Доделать (и только это):**
- Checkout €10 зашитый в продукт (Paddle/LS): демо-оценка бесплатно → 20 пакетов за €10.
- Минимальный кабинет: посмотреть/скачать свой прогон (вероятно частично есть — `/run/latest`, `/matches`, signed URLs). НЕ полноценный с сохранёнными поисками.
- Публичная read-only ссылка на разбор (вирусный контур — люди шерят в чаты).
- Деплой бэка (тут оправдан). **Триггер: переход на PR-flow + branch protection — main становится prod.**

**Метрика:** ≥10 self-serve оплат без твоего участия; конверсия демо→оплата ≥8%.

## ФАЗА 3 — Дистрибуция (4 недели)

CAC ≈ 0, по одному каналу в неделю:
- **Нед. 1:** партнёрства с админами релокационных каналов («еженедельный разбор 3 вакансий под B2B/ИП, бесплатно, с подписью»).
- **Нед. 2:** build-in-public на русском — контент = разборы вакансий = демо продукта.
- **Нед. 3:** LinkedIn на английском — восточноевропейские контракторы (поляки/сербы/украинцы с ФОП, та же боль).
- **Нед. 4:** Хабр-статья «Как проверяю вакансии на совместимость с ИП». НЕ Product Hunt (аудитория не твоя).

**Метрика:** 40 суммарных оплат, ≥30% из каналов, где не продавал лично.

---

## ФАЗА 4 — Подписка + расширения (после доказанного спроса)

Только когда one-time доказал, что платят:
- Подписка-мониторинг €9/мес поверх (сохранённые поиски → уведомления о новых вакансиях). Естественный апгрейд тёплого юзера, не холодная продажа.
- Группа «Навыки» в поиске (широкий охват, раскрывает смежные роли — твоя идея про SQL→аналитики). Опционально, выключено по умолчанию, честный ярлык «шире, но грязнее».
- Pool-read (когда юзеров достаточно, чтобы пул компаундился).

## Юнит-экономика (сверить на первых прогонах)

- Цена €10 / 20 пакетов = €0.50/пакет. COGS пакета (Haiku-скоринг + Sonnet-генерация) — центы. Маржа положительная после MoR-комиссии (~5–7%). **Проверь реальную себестоимость 20 пакетов на живом прогоне.**
- LTV: реалистично 1.5–2 покупки на юзера ≈ €15–20. Потолок CAC ≈ €5–8 → платный трафик исключён, только органика/комьюнити. Это и есть модель.
- Честный потолок ниши: русскоязычный сегмент даёт реалистично €1–3k «оборота» на горизонте, не венчур. Это плацдарм + первые деньги. Расширение — англоязычные восточноевропейские контракторы, после того как русскоязычная механика докажет конверсии.

## Сводные критерии остановки

| Веха | Должно быть правдой | Иначе |
|---|---|---|
| Конец Фазы 1 | Прогон даёт 20–40 вакансий, не ноль | Пересмотреть обещание «20 пакетов» или впервые думать о платных источниках |
| Конец Фазы 0 | 5 ручных оплат | Ниша не платит → контент/консалтинг, не SaaS |
| Конец Фазы 2 | 10 self-serve оплат | Продукт не самообслуживается → чинить онбординг |
| Конец Фазы 3 | 40 оплат, треть без тебя | Нет дистрибуции без основателя → консалтинг |

## Самое трудное (и самое важное)

Ты инженер — тебя тянет улучшать продукт вместо того, чтобы проверять, платят ли за него. Мы это видели весь этот разговор: каждый раз у развилки монетизации всплывала новая фича. Правильное действие в Фазе 0 — **почти не писать код**. 60 диалогов с живыми людьми дадут больше, чем любой коммит. Фаза 1 (охват) — последний блок кода перед тем, как идти к людям. После неё — продавать руками, а не строить.
