# ROADMAP

Дорожная карта продукта. Два трека делят общее ядро (`jobsearch/`):
личный пайплайн (рабочий) и многопользовательское приложение на той же логике.

> Статусы: ✅ сделано · 🚧 в работе · ⬜ запланировано

## Фаза 0 — чистое ядро 🚧
Вынести из CLI чистый, тестируемый API без изменения поведения скрейпинга/скоринга/тюнинга.
- `scrape(params) -> list[Job]` — все настройки приходят параметрами, не из глобалов/argv.
- `score(job, resume_text) -> dict` — Haiku-предфильтр как чистая функция.
- `tailor(job, resume_struct) -> Package` — docx-байты + cover letter + ATS-отчёт В ПАМЯТИ.
- Состояние `seen`/`processed` передаётся аргументом, не читается из `.txt` внутри функций.
- CLI (`finder.main` / `pipeline.main`) — тонкие обёртки: argv/env + I/O на краю, ядро чистое.

## Фаза 1 — структура и инфраструктура ✅
- Пакетный layout `jobsearch/`, `scripts/`, `deploy/`, `tests/`, `pyproject.toml`.
- Тесты фильтров, установка через `pip install -e .`.

## Фаза 2 — продукт (многопользовательский) ⬜
- Профиль/резюме на пользователя (мульти-CV вместо одного `master_cv.md`).
- Очередь и хранилище результатов (БД вместо CSV/файлов на диске).
- Веб-интерфейс / API поверх ядра `scrape`/`score`/`tailor`.
- Управление ключами и квотами на пользователя.

## Фаза 3 — масштаб и качество ⬜
- Прокси-пул и троттлинг под объём LinkedIn.
- Метрики качества подгонки (отклик → ответ → интервью) и A/B промптов.
- Кеширование описаний и инкрементальный сбор.
